C++技术Q&A（二）

  1. #include <xxx.h> 和 #include “xxx.h"有什么区别

    #include <xxx.h> 用于标准库头文件：通常用于包含标准库的头文件。

    搜索路径：编译器首先在标准库的目录中查找文件。这些目录通常是编译器安装时配置的目录。例如，在C++中，#include 会让编译器在其标准库路径中寻找 iostream 头文件。

    意图：这种形式表明你正在包含一个通用的、标准的或第三方库中的头文件。

    #include “xxx.h” 用于项目中的头文件：通常用于包含用户定义的头文件，即你在项目中创建的头文件。

    搜索路径： 编译器首先在当前源文件所在的目录查找文件。 如果在当前目录中找不到，编译器会在指定的包含路径中搜索（例如，使用 -I 选项指定的目录）。 最后，编译器可能会在标准库目录中查找，但这是实现定义的。

    意图：这种形式表明你正在包含一个与当前项目更紧密相关的头文件，通常是你自己编写的头文件。

    前者通常用于包含标准库的头文件。引用内容通常是一个通用的、标准的或第三方库的头文件。

    后者通常是引用我自己在项目中创建的头文件。它与项目的相关性更高。

  2. const的用途

    1. 可以定义一个常量变量，这个值在初始化之后不可修改。

    2. const int* p ,不能通过指针修改他指向的值。

    3. int* const p,不能修改它指向的地址。

    4. void func(const int& i) 函数不可修改ｉ值，避免拷贝。

    5. int func() const{} 修饰类的成员函数，保证函数不会修改类里的成员变量。

    6. const int func() 函数返回常量对象，调用者不能修改这个返回的常量对象的内容，同时这个对象也只能调用const函数。

    7. const int func() const{} 调用者不可以通过该返回的引用修改原对象。

    8. const int* func() 不能通过指针，修改原对象。

      总结：防止修改，明确意图，提高安全性。

  3. C++中调用C函数，为什么要加extern “C”？

    因为他们的函数名在编译后的符号表中有不同的命名规则。如果没有这个说明，c++ 代码无法直接链接调用c语言的函数。

  4. 链接动态库时有一个函数未找到，该如何排查解决 ？

    1. 库文件未正确链接 -lceres
    2. 函数名称不匹配
    3. 找不到正确的库所在的路径
    4. 库的版本不对
    5. 静态和动态链接混用

  5. 全局变量/函数 加不加static有什么区别?

    • 加了，具有内部链接属性，只在定义他们的文件内可见，其他文件不能用。可避免命名冲突。
    • 不加，具有外部链接属性，可通过extern在多个文件中访问。

  6. C++异常可以跨线程传播吗

    不能直接跨线程传播，可以通std::promise和std::future间接完成。

  7. 构造函数里面是否可以调用虚函数

    可以调用虚函数，但可能不会表现出多态行为。

  8. #ifndef、#define、#endif的作用

    • 他们都是预处理指令，
    • #ifndef用于检查某个宏是否未定义，如果没定义就执行。。。
    • #define用于定义一个宏，在头文件保护符中，通常用于定义一个唯一的宏。
    • #endif用于结束#ifndef的条件块。
    • 用这一套东西，可以避免头文件的重复包含和类的重复定义。

  9. 虚函数和虚表有什么区别与联系

    虚表是一个指针数组，每个元素都是指向虚函数的指针。

    当通过基类指针或引用调用虚函数时，编译器会使用虚表指针查找实际函数的地址，然后调用该函数。

    这样就可以在运行时根据对象的实际类型调用正确的函数版本。

  10. c++几种继承方式?

    访问控制符protected：类内部和派生类能访问。

    1. public inheritance

      基类的private成员，无法直接在派生类中访问。

    2. protected inheritance

      在公有继承的基础上，基类的public和protected成员，在派生类中都变成了protected.

      这种继承方式限制了外部对象对派生类和基类成员的访问，但允许派生类的子类访问。

    3. private inheritance

      在公有继承的基础上，基类的public和protected成员，在派生类中都变成了private.

  11. 多态有几种实现

    多态允许相同的操作以不同的方式执行。它有两种方式实现：

    1. 静态多态，编译时多态
      1. Function Overloading，同一个函数名可以有多个不同的实现，具体调用哪个函数由参数的类型和数量决定。
      2. Operator Overloading，运算符可以被重载为自定义类型的操作，例如重载+运算符来实现两个对象的相加。
    2. 动态多态，运行时多态
      1. Virtual Function，通过在基类中定义虚函数，并在派生类中重写它们，可以实现不同对象调用相同函数时表现出不同的行为。
      2. Pure Virtual Function and Abstract Class，如果一个类包含一个或多个纯虚函数，那么这个类就是一个抽象类，不能直接实例化。派生类必须实现所有纯虚函数才能实例化。

  12. map和unordered_map的差别，使用场景，底层数据结构

    1. 差别：前者有序，后者无序。
    2. 底层结构：前者是红黑树，所有操作的时间复杂度为O(log n)。后者是哈希表，大多数操作的时间复杂度平均为O(1)，最坏情况下为O(n)。
    3. 内存使用：map通常比unordered_map使用更少的内存，因为它不需要额外的哈希表存储。由于哈希表的开销，unordered_map通常会使用更多的内存。
    4. 选择map: 当你需要有序的键值对时，或者你需要在键的顺序上进行复杂操作，比如范围查询，map是更合适的选择。
    5. 选择unordered_map: 当你不关心顺序而更关心操作的效率时，unordered_map通常会提供更好的性能，尤其是在处理大量数据时。

  13. std::atmoic

    std::atomic 是 C++ 标准库中的一个模板类，用于在多线程环境中实现原子操作。它保证了对变量的访问是线程安全的，即无论多少线程同时读写同一个 std::atomic 对象，都不会产生数据竞争。提供了对基本数据类型（如整数、指针等）的原子操作支持，避免了使用锁（如 std::mutex）带来的开销和复杂性。

    使用场景：

    • 并发计数器：多个线程同时对一个计数器进行加减操作时，使用 std::atomic 确保操作的原子性。
    • 标志变量：在多线程应用中，使用 std::atomic 来标识某个状态或条件是否满足。
    • 无锁数据结构：实现无锁队列、栈等数据结构时，使用 std::atomic 确保操作的安全性和高效性。

  14. 重载比较运算符

    一些设计上的规则：

    1. 重载的比较运算符通常是const成员函数，因为它们不修改对象的状态。
    2. 通常应保持==与!=成对重载，<、>、<=、>=成对重载，以保证逻辑一致性。
    3. 可以通过重载其中一个运算符，再在其基础上实现其他运算符。例如，定义<后，可以通过<实现<=、>、>=。

class Point { 
private:     
    int x, y;  
public:     
    Point(int x, int y) : x(x), y(y) {}      
    // 重载 "==" 运算符     
  	bool operator==(const Point& other) const {         
        return (x == other.x) && (y == other.y);     
    }      
    // 重载 "!=" 运算符     
    bool operator!=(const Point& other) const {         
        return !(*this == other);     
    }      
    // 重载 "<" 运算符 (假设按x坐标优先排序)     
    bool operator<(const Point& other) const {         
        if (x < other.x)             
            return true;         
        if (x == other.x)             
            return y < other.y;         
        return false;     
    }      
    // 重载 ">" 运算符     
    bool operator>(const Point& other) const {         
        return other < *this;     
    }      
    // 重载 "<=" 运算符     
    bool operator<=(const Point& other) const {         
        return !(other < *this);     
    }      
    // 重载 ">=" 运算符     
    bool operator>=(const Point& other) const {         
        return !(*this < other);     
    }
